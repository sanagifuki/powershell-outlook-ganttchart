# 仕様メモ

## データファイル

- `schedules.json`: Outlook同期データのキャッシュ。
- `logs.json`: 作業ログ。
- `error.log`: Outlook同期エラーなど。

これらは `.gitignore` で除外する。

## タイトル記号

- `✕`: 絶対期限。
- `◆`: 推奨期限。
- `◇`: 目安期限。
- `▶`: 予定日。
- `★`: 参照用。

## ステータス

Outlookカテゴリーを優先する。

- `完了`: ステータスを完了にする。
- `廃止`: ステータスを廃棄にする。
- `★` かつ未完了/未廃棄: 表示。
- それ以外: 未着手。

## ガント表示

主な判定は `Domain/` 側に分離している。

- `GanttCell.ps1`: 1日1セル分の記号、背景色、ツールチップ、コーナーマーク判定。
- `GanttTaskFilter.ps1`: ガントに表示するタスクの絞り込み。
- `GanttTableBuilder.ps1`: DataGridへ渡すDataViewの組み立て。
- `GanttColumnTheme.ps1`: 日付列の背景色とヘッダー色。

## 表示フィルタ

- 未着手で終了日が `今日 + 44日` より先の予定は非表示。
- 完了は直近15件を表示。
- 廃棄は直近15件を表示。

## 作業ログ

- `logs.json` に保存する。
- 既存ログ編集時は、`uid`, `date`, `time`, `content` が一致するログを置き換える。
- 作業時間は表示時に `分` が付いていなければ付与する。

## 開発・検証

- `run-dev.ps1`: 分割ファイルを読み込んで開発起動する。
- `build.ps1`: `dist/OutlookGantt.ps1` を生成する。
- `test.ps1`: UIやOutlook実体を使わない軽量ロジックテスト。
- `verify.ps1`: 構文確認、軽量テスト、ビルド、生成物構文確認をまとめて行う。
